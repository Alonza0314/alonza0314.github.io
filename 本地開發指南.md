# 本地開發指南

## 環境需求

- Node.js: 版本 22.x 或更高
- Yarn: 已安裝（本機已具備）
- Git

## 開始開發

### 1. 安裝依賴

```bash
yarn install
```

### 2. 啟動開發伺服器

```bash
yarn dev
```

執行後，開發伺服器會在 `http://localhost:5173` 啟動（端口號可能會不同，請查看終端輸出）。

### 3. 在瀏覽器中查看

打開瀏覽器，訪問 `http://localhost:5173` 即可查看網站。

開發模式下，當您修改代碼時，頁面會自動重新加載。

## 構建生產版本

### 構建網站

```bash
yarn build
```

構建完成後，生產版本文件會輸出到 `dist` 目錄。

### 預覽生產版本

```bash
yarn preview
```

這會啟動一個本地伺服器來預覽生產版本的網站。

## 部署到 GitHub Pages

### 前置設定

1. 確保您的 GitHub repository 名稱為 `alonza0314.github.io`
2. 在 GitHub repository 設置中：
   - 進入 Settings → Pages
   - Source 選擇 "GitHub Actions"

### 自動部署

當您將代碼推送到 `main` 或 `master` 分支時，GitHub Actions 會自動：
1. 安裝依賴
2. 構建項目
3. 部署到 GitHub Pages

部署完成後，您的網站會在 `https://alonza0314.github.io/` 上線。

### 手動觸發部署

您也可以在 GitHub repository 的 Actions 頁面手動觸發部署工作流程。

## 開發注意事項

### 文件路徑

- 所有公共資源（圖片、PDF 等）應放在項目根目錄，因為它們會被 Vite 自動處理
- 在代碼中引用這些資源時，使用絕對路徑（例如：`/img/parisTu.jpg`）

### 樣式系統

項目使用 CSS 變量系統來管理顏色、間距等設計元素。所有全局變量定義在 `src/styles/index.css` 中：

- 主色系：`--primary-color`、`--secondary-color`、`--accent-color`
- 背景色：`--bg-primary`、`--bg-secondary`、`--bg-card`
- 文字色：`--text-primary`、`--text-secondary`、`--text-light`
- 陰影、邊框半徑、間距等也都有對應的變量

### 響應式設計

所有頁面都已實現響應式設計，支持以下斷點：
- 桌面：1024px 以上
- 平板：768px - 1024px
- 手機：480px - 768px
- 小型手機：480px 以下

## 常見問題

### 開發伺服器無法啟動

1. 確認 Node.js 版本是否符合要求（22.x 或更高）
2. 刪除 `node_modules` 和 `yarn.lock`，然後重新執行 `yarn install`
3. 檢查是否有其他程序占用了端口

### 構建失敗

1. 確認所有 TypeScript 錯誤都已修復
2. 執行 `yarn install` 確保所有依賴都已正確安裝
3. 檢查終端輸出的錯誤信息

### GitHub Pages 部署失敗

1. 確認 GitHub repository 設置中 Pages 的 Source 已設置為 "GitHub Actions"
2. 檢查 Actions 頁面的錯誤日誌
3. 確認所有必要的文件都已提交到 repository
